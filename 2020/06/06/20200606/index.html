[Hexschool Web Layout Training] — 八週總結
發表於 2020-06-06| 更新於 2021-06-09|Code前端Hexschool-Web-Layout-Training[六角網頁切版直播班]
字數總計: 3.2k|閲讀時長: 9 分鐘

Introduction & 前言

為期八週的切版直播班結束了，但真正的挑戰才正要開始。

如果你跟我一樣，不是在切版就是在前往切版的路上，但是想加強自己的切版能力，所以正在思考要不要購買課程，那就可以考慮看看這篇心得文。

如果已經聽過 六角學院 且購買過他們的課程，也可以看看這篇文章，說不定你會成為我的下一屆學弟妹？！

Summary & 摘要

  1. 我為什麼報名這堂課？

  2. 在這堂課我獲得什麼？

  3. 總結

我為什麼報名這堂課？

雖然第一週已經提過為什麼會參加，但總結不免還是要在回顧一次自己這八週的過程。

專案時程

沒錯！就是專案時程；先大致描述一下自己，目前筆者正在某一間專門做 LINE CRM(顧客管理) 的公司，做一些關於 LINE 上可以看到某一些官方帳號或是私人企業的活動，以及 LINE 機器人、推播、LINE 後台…等等，大概就是跟 LINE 有關係的都會做到。

為什麼提到這個呢？其實我們有自己的後台供顧客使用沒有錯，但其實客戶都會想要 客製化，所以常常在某些節日，抑或是企業的行銷申請到一大筆經費要搞活動的時候，就是我們 發大財倒大楣 的時候。

這時候除了工程師自己的工作時程要拿捏好之外，PM(註1) 也一樣重要！

會提到這個是筆者個人覺得 PM 並沒有真的把時間安排得很好，且大部分都是客戶說什麼功能，就接什麼功能，然後再丟給工程師請工程師盡量能做到，由於筆者還很菜，而且是的我是奴工，不敢打倒資本主義的 高強，所以只能盡量去做了。

廢話說太多，下面我們會繼續說；因為有前因，所以後果就是筆者必須加強自己的切版能力，我想提升自己的切版能力後，總能一個打十個吧？

誤打誤撞

就這麼恰巧，六角學院就推播了一則訊息來，又是剛好關於切版，所以想了幾分鐘後就報名了，這幾分鐘想什麼呢？是關於 實體 或是 線上課程 的差別嗎？錯！是想我的荷包君夠不夠力；關於 實體 或 線上課程 的差別後面會提到。

註1: PM最常指的職位大致上可分為兩種，專案經理(Project Manager) 和 產品經理(Product Manager)，前者為規劃整個大致計畫流程、時間、架構，更重要的就是協調 產品經理(Product Manager) 去做任務的職位；後者大概就是接受前者任務然後負責追進度的！有興趣可以參考這 5 種角色都叫 PM！專案經理、產品經理，差在哪？

在這堂課我獲得什麼？

這堂課程其實滿特別，以往都是 Udemy 購買課程，然後獲得影片，在隨自己的進度去看，然後往往都沒看完，或是看一兩片就沒然後了…

這堂課開頭就寫明是直播，而且上限只收 200 個人，因為每週一堂課，且校長每週都會出作業，然後可以透過他們的後台系統提交，所以為了改程式碼的品質，就收 200 為主。

在課程開始前校長就請大家先自我介紹然後幫大家分組，且請大家加入專屬的 Slack，所以大家除了共用頻道外，還有各個小組的頻道，每週除了個人作業外，還有小組作業，所以才會分組，目的除了解任務，最主要我覺得還是透過彼此加油打氣讓自己更有動力往前進！

分組的原則基本上是藉由自我介紹來分組，會將差不多性質的同學歸再一起，比如說 設計師、學生、轉職者…等等。

這次筆者被分配到 鞋鋼鍊金2組，其實剛開始筆者覺得應該是要被分配到 軟工盾坦組(適合以前有做過程式工作，或是現職程式領域的勇者)，但自己分到的這組同學都太給力了，最後就不小心達成了一次作業率繳交第一的成績。

分組結束就是每週五的上課，然後從八點開始一路到十點半，有時候會超過，雖然我們是消費者心態，超過就是爽，賺到了，但其實你也知道 關於注意力的研究顯示，人能維持專注的時間約在10到18分鐘，過了18分鐘後，人的身體因為能量耗盡…ballaballa(詳細可參考 每次開會都落落長？美研究：最多●人、最長●分鐘，最能保持專注)，所以後面常常上課上到出神，但還好課程跟大部分線上課程一樣都還是可以參考重播影片。

而且上課同學都很熱心，甚至還有自發性的 義工團，他們 替重播影片加上重點時間、寫課程範例、提供能力內的指導…等等，根本就是 上班族福音！

除了上方提到的 分組、重播影片 外，這次六角學院還準備了很特別的 BOSS系統，每週提交作業都能對 BOSS 造成傷害，算是挺有意思的！

講完一些廢話要進入廢話重點了，就是課程本身，由於這次接版直播班是從０開始的，所以課程很多地方都是從基礎開始說，不怕銜接不上，因為筆者也想把基礎打好，所以這次課程對我來說很剛好！

上課除了老師單方面的講解外，隨時都能發問。雖然講解者(校長)本身可能會因為被頻道文字洗掉問題，但都隨時可以在 Slack 上對校長或是助教發問。

偷偷說助教真的人超好啊，而且直播也講解的淺顯易懂！

此外作業批改的時間也都算是很迅速，幾乎都是在提交後的最晚三四天內會收到回饋，而且也不是隨便看一看，是真的有用心在回覆，我都擔心校長及助教的肝了…

最後要提的大概就是最狂的 Code Review 升級版 Video Code Review 了！校長除了看你的程式碼外，還鼓勵學生錄下自己切版的影片過程給他，然後他會給你更多建議。雖然通常老王賣瓜都會自誇，但誠如校長所說，還真沒見過課程老師會願意用那麼多時間，甚至假日的時間都在批改且觀看學生的程式碼，光這點就值得滿分了。

你說！老師都這麼用心了，你還有什麼藉口不努力認真？

業配？

前面講了一堆好處，你可能會以為我在業配，雖然我也很想，但筆者為人秉持的原則是實話實說，所以這邊也要來說說筆者認為的幾個缺點(希望不會被黑名單

線上課程除了大家都知道的比起實體課程會較沒推動力，容易懶惰外，這點其實還算可以放心。

前幾週校長有提到每週上課都需要簽到，如果沒有簽到校長會請專員或是親自打電話萊督促你上課，但我想既然你都來上課了，沒有點自制力，最大問題還是自己。

因為筆者自己能力不夠別人好，所以常常覺得是不是會落後別人，所以每週課程結束就會休息一天然後接著做作業，以前幾週來看幾乎都是可以完成的，且難度適中。

不過這邊說好了要說缺點，大致上就是課程或許算是比較趕，所以大概在第三週加入 RWD 後突然難度倍增，可能許多人會切到有點心灰意冷，但其實校長希望大家都能慢慢來，觀念正確最重要，所以交作業期限並非一個禮拜為限。

而 BOSS系統 我覺得似乎真的是比較娛樂性，和分組一樣，組員們並沒有真的到非常熱絡，大概就是每週討論一次，此外筆者認為小組跟 BOSS 也許能有更多的互動，比方說 支線任務，
雖然 BOSS 並非上課重點。

以上大概就是筆者覺得的缺點，但似乎也不算什麼缺點，筆者依舊認為上課最主要還是自己的態度為主，如果自己都不鞭策自己，那別人也幫不了你。

線上課程值得嗎？

上面談完了優缺點之後，就該思考線上課程值不值得，筆者認為線上 及 實體 肯定都各有優缺點，但如果今天你的情況符合以下幾點，那你就可以不用考慮可以直接拿出錢包了：

  1. 較無法抽出固定時間

  2. 需要有品質保證，不是隨便的三流教師來上課的課程

  3. 想要擁有一大批一起努力的同溫層同伴們

  4. 錢包受得了

最後一點肯定要解釋一下，不是課程貴，而是你買了一堂後你會欲罷不能想再買下一堂，這大概就是 六角學院 的魔力吧！不信你試試看。

我不敢說 六角學院 的課程跟其他課程比起來一定是最好的，但我能保證在 用心、態度、師資…等等 你都不用擔心。

完蛋了更像業配了…

上面有提到筆者通常會在課後休息一天在開始動工，這也是最近筆者一直想到的，工作、學習、放鬆，這三件事情就應該要剛好像是 width: calc(100% / 3); 那樣剛好各佔 1/3 等份，這樣生活才能達到平衡。

有好的學習才能應付工作，有好的工作結果才能夠好好放鬆生活，千萬不要讓任何一邊比重超過，你的生活會慢慢出問題。

因為好多工程師同事及朋友都因為工作而失去動力，筆者認為最理想情況就是工作及興趣合二為一，現實狀況或許很難，但相較從興趣去找尋理想的工作，筆者認為更適合從工作中尋找興趣，帶給你的不只會是成就感，也會是滿滿的經驗，甚至可能會成為你的武器，而你的生活也會過得更快樂，不是一部工作機器。

總結

我想很多人比較喜歡看總結，所以這邊就簡單的來總結一下這們課程的八週心得。

  1. 門檻低好入門。

  2. 講解清晰易懂，老師助教有問必答。

  3. 不脫離主題之外，課程設計有趣及多元化。

  4. 有一大批的熱情六角同學隨時陪你奮鬥，這條路上不孤單。

  5. 校長課程大放送！

第五點其實不是每個課程都有啦，剛好這次的直播課程結尾校長為了讓各位更有前進的動力，所以推出了很多完成幾項作品會有什麼獎勵，其中不乏更多的免費課程影片或課程折價，甚至可以加入菁英頻道，或許之後的每一部關於這個 直播班(第二屆、第三屆…) 的加碼 DLC 影片🤫。

Conclusion & 結論

上方總結提到的就是個人簡短課程心得，接下來筆者會繼續在前端的路上邁進，可能偶爾偷偷摸個後端，但還是希望大家除了在變強的道路上一起加油外，切勿忘了生活的比重，好好的工作，也好好的陪伴家人！致正在努力的你/妳。

連實體獎品都有了，真的超級用心！
