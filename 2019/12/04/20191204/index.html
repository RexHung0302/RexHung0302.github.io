[Life Note] - LINE TAIWAN TechPulse 2019

發表於 2019-12-04 更新於 2025-02-21 分類於 Code ， Note ， LINE
文章字數： 3.9k 所需閱讀時間 ≈ 4 分鐘

Introduction & 前言

其實從接觸資訊相關工作開始至今一年又25天，也是自己的第一份程式語言工作，一直都沒有去參加過什麼大型的會議，再加上自己工作的內容跟 LINE 息息相關，一直覺得自己應該去參加個資訊相關活動，很開心的意外在2019年尾聲有幸參加了 LINE TAIWAN TechPulse 2019。

這次的開發者大會是第四屆，並且首次移動到了 臺北和平籃球館，聽說規模是去年的兩倍，將近千人到場參與，因為這是我第一次參加，就不會去比較今年有多好還是多壞了。

嚴重提醒：本篇也是以筆記及心得為主的文章，如果內容有誤，敬請不吝社指出也請多多海涵，感謝各路大神！另外圖片會有模糊問題因為當天臨時靠手機拍攝，離得也較遠，還請各位大神見諒。

本次重點

今天的主題其實一直圍繞著最近很火紅的 AI人工智慧，並且延伸至 數據分析及使用、資安、公司開發規劃 及 工商時間…等等。

與以往一樣分別為 上半場 及 下半場，上半場為 LINE台灣公司員工 介紹 公司目前近況 及 未來目標，還有 Clova Chatbot AutoML 專家 - 李在元(Jaewon Lee) 及 補充說明的 LINE台灣資深資料工程師 - 孫韻如(Penny Sun)，及 國際頂尖白帽駭客 - 李丞鎮(Beist)，最後還有 LINE貼文串推薦系統(Timeline Post Recommender System) - 李知洪(Jihong Lee)…等等，下午多為開發應用示範及 LINE 內部員工介紹各項功能還有預期要上線的功能。

-> 簡單給個重點：

  1. LINE平台技術發展與最新工具

  2. 跨海大神的演講

  3. 五大專家 十大開發團隊 交流交流

  4. LINE 預期願景

報到

順帶一提這次到會場騎乘的是 GOShare 的 GOGORO，不得不說方便是很方便，但是發動了很久，原來不管是 側柱、中柱 甚至 車廂 只要一個沒關好沒收好就不能發動。

到會場會給先發一則座位的訊息，接著再到報到區，他們會給你對應顏色手環。

活動開始

上半場

第一位(LINE台灣資深技術總監 - 陳鴻嘉)

從全球與地區的兩種角度，了解 LINE 的技術策略布局；了解 LINE TW 的工程團隊如何與總部及其他國家團隊分工合作，並培養自我職能，貢獻價值。

主要提 八大人工智慧的應用領域(Speed, NLU, Voice, Data, Video, Face, OCR, Vision)

2020目標繼續朝 人工AI 發展，LINE Brain。

第二位(LINE台灣董事總經理 - 陳立人)

講者一開始就提到 Life of LINE，並且深深地提醒了使用者，你的日常生活都與 LINE 息息相關，所以他們希望能將使用者的生活上每一個細節都與 LINE 串接起來。

講者接著提到將會透過兩大策略來優化使用者體驗：

第一策略 - 一致性(consistency)：

  1. Life Of LINE

  2. LINE Mini APP

第二策略 - 透過資料科學與人工智慧的技術來優化 UX：

  1. AI Company(AI DJ, 檢舉審核系統, Smart Channel, LINE AI Hackathon(黑客松)自動報帳)

這位講者的結論是 AI, Privary, Data, Governamece

第三位(LINE 技術專家/技術傳教士 - Evan Lin)

Clova Chatbot Builder 是一項 AI 服務，讓使用者建立自己業務領域的聊天機器人，透過深度學習與自然語言處理技術建構智能對話。本議程將介紹 Clova Chatbot Builder 架構的 pipeline、如何處理對話情境資料，以及串接 LINE 平台。此外，也將展示 Chatbot Builder 的主要功能與應用案例。

講者一開始就提到去年的整年度表現 及 新的一年的時程，關於目前及未來的展望：

  1. UX/DX

  2. OA 2.0

  3. Flex Message Simulator(不需要再面對JSON)

  4. Rich menu Batch API(後台一些數據可用 API 拿到，不需要再去後台盯著)

隨後也提到了 LIFF V2 可以在手機以外的網頁上開啟並且做一些 API 的串接，以往的 LIFF 是以 LINE 開頭的，V2 改為 https 開頭。

結論：更好的使用者提現 及 開發者體驗，Flex Message V2 不用再面對 JSON，Rich menu Batch API。

AI ChatBox：

  1. OA

  2. Task-oriented

  3. Chit-chat

第四位(Clova Chatbot AutoML 專家 - Jaewon Lee 及 LINE台灣資深資料工程師 - Penny Sun)

應該是算第四組上場的是來自韓國的 Clova Chatbot AutoML 專家 - 李在元(Jaewon Lee)，隨後還有 LINE台灣資深資料工程師 - 孫韻如(Penny Sun)。

開場是以 Chatbot 為開頭，其中最主要的主菜當然是 Clova Chatbot Builder，其中提到了當未來企業要製作一個 AI 聊天機器人，甚至要開發一個內容管理系統，建立一個介面，讓使用者輸入問題並且不是關鍵字式回覆，這些 Clova Chatbot Builder 都幫你做好了，未來也會靠你的客群們來搜集訓練材料，並且慢慢成長為更精準更到位的 AI Chatbot。

不過中文博大精深，中文還未引進，看起來中文要精準的回答還需要一哩路呢。

第五位(國際頂尖白帽駭客 - 李丞鎮(Beist))

本議程將介紹企業資安團隊的運作模式，並說明企業面臨的資安風險與因應策略。此外，將介紹 LINE 資安團隊如何與開發人員合作，進行程式碼審核、檢查內部和外部網路安全，以及使用機器學習檢測惡意行為。

來自韓國，目前擔任 LINE GrayLab資安研究室 的負責人 李丞鎮(Beist)，是亞洲首位進入 世界駭客大賽DEFCON CTF 決賽的選手，也在韓國駭客競賽中獲得多次優勝；開場以 Why Enterprise Security Is Hard ? 切入。

講者在演講期間多次提到安全把關你我做起，破解工程師比破解伺服器容易。

直得一提：LINE GrayLab特別構思出Becks (資安社群聚會），以定期巡迴的方式與各地的駭客們交流最頂尖的防護技術，全球首場Becks (資安社群聚會）。相關訊息可點擊 傳送門

第六位(Timeline Post Recommender System - Jihong Lee)

LINE 貼文串讓使用者們分享貼文進行互動，取得的使用數據讓我們能為使用者提供個人化推薦服務。本議程將介紹我們如何為使用者提供完善的體驗、持續進化推薦系統，並從資料分析的角度探討相關洞見，以及如何提升整體系統品質。

講者主要在分析推薦系統 Recommender System，以及如何分析使用者然後利用演算法去推薦使用者他想看的東西，其實也是離不開 AI。

總結：Recommender System, Pareto Optimality

另外現場還 show 出了用 Timeline Post Recommender System 技術做出來的照片牆，只要現場拍照就可以上傳到照片牆，大家都拿手機出來拍照上傳，最後也被拿來當 See You Next Year 的背景圖，很有創意！

LINE PAY

緊接著都是 LINE 公司的相關技術介紹，首先登場的是 LINE PAY，介紹一些變動、 LINE BANK、APP2APP 及 API V3…等等，甚至未來出國能直接在國外使用 LINE PAY 付款，無需換匯。

LINE SPOT

再來是 LINE SPOT，他們的初衷是目前 LINE 有了眾多服務，但是卻無法好好的串起來直接使用，比方說想約朋友吃飯，但是卻要在眾多的 APP 來回穿梭，達到 訂餐廳、看菜單、叫車…等等的最終結果，因為這樣，所以 LINE SPOT 誕生了。

Fact Checker

這個講者提到，因為每天都有數已難記的檢舉數量，光靠人工是無法快速且有效率的去分析這些資料是不是假消息，所以一樣透過演算法及AI人工智慧，達到機器人分工的效果。

Other

其他還有提到收尋、音樂、QA、FCU 甚至 LINE的環境架構…等等，這邊就不一一補充說明了。

Conclusion & 結論

這次去 LINE TAIWAN TechPulse 2019 算是收穫頗多，也聽了很多家企業的分享，常常覺得自己在碰到資訊相關知識時，自己就好像走入了一片森林，自己只看到最前面那棵樹，不過就跟運動一樣，這種事情就是什麼都不重要，重要的是你的毅力呀！

最後用一張紀念品圖來結束這回合囉，以上資訊如有誤還請多見諒，圖片模糊再次跟各位大神道歉，因為很臨時的用手機拍攝，距離也較遠，當天電腦恰巧又沒電，當下都是手寫筆記，如果之後有再想起什麼片段，會再更新並且補上來。
